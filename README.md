# milk
.
漢字よみクイズのシンプルな静的ウェブアプリケーションです。ブラウザで index.html を開くだけで動作し、150問からランダムに10問を出題して漢字の読みを答える形式のゲームです。

## 主な特徴

- 単一ファイル実装：HTML/CSS/JavaScript が主に `index.html` にまとまっています。
- 問題データ：`QUESTIONS` 配列に約150問の問題を { display, correct, wrong } 形式で定義。
- セッション：`SESSION_COUNT = 10`（ランダム10問、重複なし）
- UI：スコア、連続正解（STREAK）、ライフ、進行（PROG）を表示。選択肢は縦に3行表示。
- クライアント専用：サーバ不要で静的ホスティング（例：GitHub Pages）に適しています。

## ファイル構成（主なもの）

- `index.html` - アプリ本体（UI, スタイル, 問題データ, ロジック）
- `README.md` - このファイル

## 実装の要点

- 問題データは `QUESTIONS` 配列に格納され、各要素は次の形式です：

  { display: "漢字", correct: "よみ", wrong: ["誤答1", "誤答2"] }

- セッション生成：`buildSession()` で `QUESTIONS` をシャッフルし、先頭から `SESSION_COUNT` 個を抽出します。
- 表示と操作：`renderHud()` で HUD を更新し、各選択肢ボタンにクリックイベントを登録して `judge()` で正誤判定を行います。誤答でライフが減ります。
- 補助機能：`shuffle()`, `randInt()`, `showToast()`, `setButtonsEnabled()`, `fitChoiceFont()` などのユーティリティを利用しています。

## 実行方法

1. リポジトリをクローンまたはダウンロードします。
2. `index.html` をブラウザで開くだけで動作します。
3. 必要なら GitHub Pages にデプロイしてホスティングできます。

## 変更提案・改善案

- 問題データを外部 JSON ファイルに分離して管理すると編集が容易になります。
- 正答率やランキングの保存にはローカルストレージや簡易サーバの導入を検討できます。
- モバイル表示やアクセシビリティ（キーボード操作、スクリーンリーダー対応など）の改善。
